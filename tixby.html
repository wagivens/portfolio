    • Tixby

    December 2020

    A simple, elegant solution for purchasing tickets to events.

    The idea for this project came from a prompt on Designercize that focused on musicophiles searching for live events. Immediately, I went to work on designing a few screens.

    When I finished, something did not feel right. I thought the user interface felt amateur and rushed (even though these prompts are typically meant for quick ideation and prototyping).

    So I decided that I would redo things, and that I would just create an entirely new product/application instead of only a couple of screens; I would use it as an opportunity to genuinely practice my user interface design skills.

    Project Summary

    My goal for this project was to simply create a visually attractive, intuitive interface. My process was more condensed than it would be for a larger, established product. Three separate phases covered Project Definition, Ideation, and Prototyping, eventually culminating in a finished mockup.

    Phase 1: Defining

    I began the project by doing a simple competitive analysis to better define the product parameters. At the moment, Ticketmaster and AXS Tickets are the frontrunners in live event ticketing; StubHub and SeatGeek are of similar popularity among ticket resellers. I downloaded their applications and studied their user flows and interactions as I went through them.

    Of course, there was no single competitor that had every aspect I believed was most beneficial for users. Still, I noted elements and features from each application that made navigating through them easy for me: large banners for trending events, letting users view events by category, and a header on the home page which featured a search bar and results filters.

    Phase 2: Ideation

    Next was a product feature/screen roadmap. I usually like to outline which elements I think should be on each screen, but with minimal sketching. This prevents me from getting stuck when actually sketching.

    Phase 3: Prototyping

    I wanted minimal color for this product to avoid complicating the user interface. Of course, there are neutral Grays and Whites for backgrounds, elements, and some text. I settled on #59A9F5 as Tixby's primary color based on color psychology; blue usually represents trustworthiness and comfort, and those are both important qualities for a ticket-buying platform.

    I then embarked on creating high-fieldity mockups of my sketches. I kept the number of screens minimal (since this was just practice), but wanted to show the process of entering the app, filtering out search results, selecting an event, and purchasing a ticket for it.

    Reflecting and Next Steps

    Overall, I enjoyed working on this project! If this were an established product, I of course would go on to test out my solution and iterate on the designs. In any case, this was great practice for building user interfaces and incorporating many UI standards.
