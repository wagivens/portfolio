Messages Edit Feature

  • Intro
  • Ideating
  • Prototyping
  • Reflection

It is surprising that Apple's Messages application has no edit feature since users can already interact with texts in multiple ways. The absence of an edit feature means users sometimes resend messages with corrections, cluttering their text threads. This made me curious as to what an edit feature in Messages would look like.

Starting Where We Are

In referencing the existing Messages interface, I noticed the "Tapback" feature uses a corner icon to clearly indicate a message was interacted with. I considered having a unique icon at the bottom of any message that was edited.

Moreover, clicking a message reply blurs the interface while focusing on the original message and its reply thread. I thought something similar could be done for edited messages; essentially, an “edits” thread.

In general, I figured modeling the edit feature after existing ones would make it both technically feasible for Apple's software engineers, as well as familiar for Messages users.

Thinking Through the Solution

I first thought about which icon(s) would best represent the concept of editing and ultimately decided on a pencil, which can already be found throughout iOS; it is typically used to indicate a user can change or create a text-based component (ex: altering a note in the Notes application, creating a new text message thread, writing a new email, etc).

Color was also important. I determined Apple's “System Yellow” would best signify a component had been changed or was being worked on. Apple often uses System Yellow when something is being altered (ex: editing pictures in the Photos app, changing settings in the Camera app, etc). In the physical world, yellow is sometimes used for alerts or to denote "work in progress."

Prototyping

I then put everything together. Using an iOS UI kit on Figma, I prototyped what I believed were the two most important user flows: editing a message and viewing message edits.

Reflecting

One of my next steps for this would be to optimize for accessibility. Some of my design decisions were rooted in the color of components, but many users may not be able to interpret them. As such, it would be beneficial to figure out how to convey the same ideas without relying on color.
